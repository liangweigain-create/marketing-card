# 帮助文档配置指南

本文档介绍如何维护和更新网站的帮助中心。

目前系统已升级为**全配置驱动**模式，您**不需要懂任何代码**即可完成文档的增删改查。

---

## 📂 核心文件位置

只需要关注这两个地方：

1. **配置文件**：定义目录和文章结构
   👉 `src/pages/help/config/help.config.ts`

2. **内容文件夹**：存放具体的文章内容（Markdown 格式）
   👉 `public/content/help/[分类名]/[文章ID].md`

---

## 🛠 常见操作指南

### 场景一：添加一篇新文章

假设您想在「管理员指南」下添加一篇名为「如何导出报表」的文章。

#### 第一步：修改配置文件

打开 `help.config.ts`，在 `HELP_SIDEBAR` 中找到 `manager` 分类，添加一行配置：

```typescript
// help.config.ts
manager: [
  {
    title: '数据分析',
    items: [
      // ... 原有文章
      { label: '如何导出报表', slug: 'export-report' } // 👈 新增这一行
    ],
  },
],
```

- `label`: 侧边栏显示的标题
- `slug`: 文章的唯一标识 ID（只能用英文和连字符）

#### 第二步：创建内容文件

在 `public/content/help/manager/` 目录下，创建一个名为 `export-report.md` 的文件。

#### 第三步：编写内容

打开刚才创建的 `.md` 文件，写入内容：

```markdown
---
title: 如何导出报表
readTime: 3
updatedAt: 2026-01-20
---

# 导出报表流程

这里是正文内容...

## 步骤说明

1. 进入后台
2. 点击导出按钮
```

**🎉 完成！** 刷新页面即可看到新文章。

### 场景二：添加一个新的文档分类

假设您想新增一个「开发者文档 (developer)」板块。

#### 第一步：注册分类

打开 `help.config.ts`，在 `HELP_CATEGORIES` 数组中添加：

```typescript
{
  id: 'developer',        // 分类 ID
  title: '开发者文档',     // 显示名称
  description: 'API 接口集成指南',
  icon: Code,             // 图标（需从 lucide-react 导入）
  iconColor: 'text-green-600',
},
```

#### 第二步：配置侧边栏

在 `HELP_SIDEBAR` 中添加该分类的目录结构：

```typescript
export const HELP_SIDEBAR = {
  // ... 其他分类
  developer: [
    {
      title: "接口指南",
      items: [{ label: "快速开始", slug: "quick-start" }],
    },
  ],
};
```

#### 第三步：创建目录和文件

1. 创建文件夹：`public/content/help/developer/`
2. 创建文件：`public/content/help/developer/quick-start.md`

**🎉 完成！** 系统会自动生成 `/help/developer` 页面和入口卡片。

## 📝 Markdown 编写技巧

我们的文档支持丰富的格式：

### 1. 基础格式

```markdown
# 一级标题（页面标题）

## 二级标题（章节）

### 三级标题

**加粗文字**
_斜体文字_

- 列表项 1
- 列表项 2
```

### 2. 添加图片

将图片放入 `public/content/help/[分类]/images/` 文件夹，然后引用：

```markdown
![图片描述](/content/help/manager/images/step1.png)
```

### 3. 提示框

引用语法会被渲染为漂亮的蓝色提示框：

```markdown
> 💡 提示：导出数据可能需要 3-5 分钟生成时间。
```

## ❓ 常见问题

**Q: 修改了配置文件，页面没更新？**
A: 一般会自动刷新。如果没变，请尝试刷新浏览器。

**Q: 侧边栏点击没反应？**
A: 请检查配置文件中的 `slug` 和 Markdown 文件名是否完全一致（注意大小写）。

**Q: 文章里的图片不显示？**
A: 请确保图片路径以 `/content/help/...` 开头，不要用相对路径（如 `./images/`）。
